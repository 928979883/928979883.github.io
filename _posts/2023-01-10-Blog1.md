## 第一步：在 GitHub 创建仓库

1. 打开 [GitHub](https://github.com/) → 点击 **New repository**。
2. 仓库名必须为：`你的用户名.github.io`（`username` 即你的 GitHub 账号名）。
3. 仓库需保持 **Public**（GitHub Pages 用户页要求公开）。
4. 勾选 **Add a README file**，点击 **Create repository**。

> 💡 **提示**：这是「用户/组织页」，GitHub 会自动把 `master` 分支根目录当作网站根，无需额外配置，比「项目页」更简单。

验证是否生效：创建后等 1–2 分钟，访问 `https://你的用户名.github.io`，能看到 README 即说明第一步成功。

## 第二步：搭建本地环境（Ruby + Jekyll）

Jekyll 基于 Ruby，需要本地安装后才能预览调试。

### 安装 Ruby（Windows）

1. 前往 [RubyInstaller 下载页](https://rubyinstaller.org/downloads/)，选择 **WITH DEVKIT** 版本。
   > ⚡ 小技巧：用迅雷复制下载链接下载，速度更快。
2. 安装时务必勾选 **Add Ruby executables to your PATH**。
3. 安装完成后会自动进入 MSYS2 安装，输入 `1` 回车即可。
4. 验证：`ruby -v` 能看到版本号即成功。

### 安装 Jekyll 与 Bundler

```bash
# 验证 gem
gem -v

# 国内建议先切换 gem 源，速度起飞
gem sources --add https://gems.ruby-china.com/ --remove https://rubygems.org/

# 安装 bundler 与 jekyll
gem install bundler jekyll

# 验证
bundle -v
jekyll -v
```

> ⚠️ **注意**：若 `gem install` 卡住或超时，基本都是网络问题，切换上面的 RubyChina 镜像源通常能解决。

## 第三步：获取博客模板

你可以从零写，但更推荐直接 fork 一个现成主题改造：

- **LOFFER**：轻量、适合写作的中文 Jekyll 主题 → [预览](https://fromendworld.github.io/LOFFER/)
- 本文博客模板：已集成搜索、统计、暗色模式，可直接 clone 改造

拿到模板后，把仓库内容替换为你自己的 `用户名.github.io` 仓库即可。

## 第四步：本地预览

进入博客根目录执行：

```bash
# 安装依赖
bundle install

# 启动本地服务（默认 http://localhost:4000）
bundle exec jekyll serve
```

> 💡 想指定端口：`bundle exec jekyll serve --port 10240`，然后访问 `http://localhost:10240`。

修改文件后会自动热更新，刷新浏览器即可看到效果。

## 第五步：核心配置（`_config.yml`）

站点的全局信息都在这个文件里。常见配置项：

```yml
title: 你的博客名
description: 一句话介绍
url: https://你的用户名.github.io
author: 你的名字

# 社交链接（用于首页与页脚）
footer-links:
  github: 你的用户名
  email: you@example.com
  zhihu: 你的知乎ID
  weibo: 你的微博ID
  rss: true
```

> ⚠️ **安全提醒**：如果配置评论系统（如 Gitalk），**切勿把 `clientSecret` 提交到公开仓库**。本博客已弃用评论区以规避此风险。

## 第六步：增强功能

### 6.1 全文搜索

依赖 [Simple-Jekyll-Search](https://github.com/christian-fei/Simple-Jekyll-Search)：

1. 在根目录创建 `search.json`（定义索引字段：标题、标签、日期等）。
2. 把 `simple-jekyll-search.min.js` 放到 `js/` 目录。
3. 在首页 `index.html` 加入搜索框与初始化脚本：

```html
<div class="search-container">
  <input type="text" id="search-input" placeholder="搜索文章...">
  <ul id="results-container"></ul>
</div>
<script src="{{ site.baseurl }}/js/simple-jekyll-search.min.js"></script>
<script>
  window.simpleJekyllSearch = new SimpleJekyllSearch({
    searchInput: document.getElementById('search-input'),
    resultsContainer: document.getElementById('results-container'),
    json: '{{ site.baseurl }}/search.json',
    searchResultTemplate: '<li><a href="{url}">{title}</a></li>',
    noResultsText: '没有找到相关文章',
    limit: 10,
    fuzzy: false
  });
</script>
```

![search_block]({{ site.baseurl }}/images/2023-01-10-Blog1/search_block.png)

### 6.2 访问统计

**51.LA**（功能丰富，含来源、地域分析）— 注册后在 `</head>` 前加入：

```html
<script charset="UTF-8" id="LA_COLLECT" src="//sdk.51.la/js-sdk-pro.min.js"></script>
<script>LA.init({id:"你的统计ID", ck:"你的统计ID"})</script>
```

![access_statistics]({{ site.baseurl }}/images/2023-01-10-Blog1/access_statistics.png)

**不蒜子**（轻量 PV/UV）— 在需要显示的页面加入：

```html
<script async src="//busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"></script>
<span id="busuanzi_container_site_pv">浏览量 <span id="busuanzi_value_site_pv"></span></span>
```

> 💡 两类统计都建议 **异步加载**（`async`），避免拖慢首屏渲染。

### 6.3 文章目录（TOC）

在文章 Front Matter 中加一行即可自动生成目录：

```yml
---
layout: post
title: 文章标题
toc: true
---
```

### 6.4 主题色与暗色模式

本模板使用 [Open Color](https://yeun.github.io/open-color/) 色板。修改 `_sass/_variables.scss` 中的主色变量（如 `grape` → `indigo`）即可换肤；暗色模式由 CSS 变量 + 一键切换脚本实现。

## 第七步：写你的第一篇文章

1. 所有文章放在 `_posts/` 目录。
2. 命名格式必须为：`YYYY-MM-DD-文章名.md`（例如 `2026-07-22-hello.md`）。
3. 文件头（Front Matter）示例：

```yml
---
layout: post
title: "我的第一篇博客"
date: 2026-07-22
tags: [life]
toc: true
---
```

4. 正文用 Markdown 编写，`<!-- more -->` 之后为列表页摘要之后的内容。
5. `git push` 后，GitHub Pages 会自动构建并发布。

## 🛠 常见问题

**Q：提示缺少某个 gem（如 `jekyll-paginate`）？**
A：缺什么装什么：`gem install jekyll-paginate`，或检查 `Gemfile` 是否声明了该依赖后执行 `bundle install`。

**Q：`Permission denied` / 端口被占用？**
A：默认 4000 端口被占。查找并结束占用进程：

```bash
# 查找占用 4000 端口的 PID
netstat -ano | findstr "4000"

# 根据 PID 查看进程名
tasklist | findstr 进程PID

# 然后到任务管理器结束该进程，或换端口启动
bundle exec jekyll serve --port 5000
```

**Q：推送后页面没更新？**
A：GitHub Pages 构建需要 1–3 分钟；可在仓库 **Settings → Pages** 查看构建状态与错误日志。
